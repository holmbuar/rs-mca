## Purpose

The C7 packet proves one deletion-aware local producer but does not close the
semantic atlas or `(UNIF)`. The current authority names two adjacent hard
inputs:

1. Sidon/Fourier payment or equivalent effective-image MI+MA / row-sharp
   max-fiber control on the actual primitive residual;
2. a residual ray compiler for higher-dimensional balanced cores.

This packet tightens the finite producer boundary for one scoped C8 route and
one scoped C9 route. It deliberately does not modify PR #997.

## 1. Shared one-profile line extension

`AsymptoticSpine.ClosedLedgerExtension` defines the flattened slope image of an
already closed line and appends one `ProfilePayment` only after an explicit
cross-owner disjointness proof. The construction proves:

```text
new assigned slopes = earlier assigned slopes ++ profile assigned slopes,
new budgetTotal     = earlier budgetTotal + profile.rayBudget,
new naturalTotal    = earlier naturalTotal + profile.naturalScale.
```

The output preserves `ClosedLineLedger.firstMatchOwnership`, numerical atlas
exhaustiveness for the enlarged covered numerator, and the profile-count cap.
It is a compiler lemma, not a classifier or payment theorem.

## 2. C8: shallow-prefix residual producer

### Input object

`C8ShallowClosure.ProfileData` contains:

- an actual `PrefixFiberBridge` with the Newton/affine reindexing guard;
- one syndrome key;
- an `(SE2)` certificate for the supplied distinct slopes and support
  projection;
- proof that the support projection is a sublist of the reindexed ambient
  prefix chart;
- `ShallowPrefixClosureBounds`; and
- the printed residual-kernel label.

### Proved payment

The already formalized theorem
`balancedCoreShallowClosure_to_directRC` gives

```text
slopeCell.slopes.length
  <= (baseSize ^ prefixDepth) * (1 + average)
```

for every kernel-dimension label. The new adapter installs this as

```text
ProfilePayment.ofDirect .c8
```

with natural scale `1 + average` and compiler loss
`baseSize ^ prefixDepth`. It then appends the profile to an earlier line using
an explicit slope-disjointness proof.

The executable fixture uses kernel label `1000000`, earlier C1 slope `5`, and
C8 slopes `[7,9]`. Its exact totals are:

```text
assigned slopes = [5,7,9],
budgetTotal     = 16 = 4 + 12,
naturalTotal    = 4  = 1 + 3,
badCount        = 3.
```

### Scope boundary

This closes only the typed path

```text
supplied shallow residual chart
  -> SE2 support injection
  -> kernel-independent shallow direct RC
  -> direct C8 ProfilePayment
  -> line-local closed-ledger extension.
```

It does not prove chart exhaustion, actual common-factor size, the shortened
row dimension for every survivor, deep-prefix MI/MA, or a structure-sensitive
large-kernel theorem outside shallow closure.

## 3. C9: exact post-C1--C8 max-fiber producer

### Specification repair

A bare `PrimitiveBooleanLeaf` records a complete fixed-weight slice and an
arbitrary residual sublist. It does not make the phrase "surviving C1--C8"
evaluable. `C9ResidualMaxFiber.ProfileData` therefore adds:

```text
earlierOwner : support -> Option PreC9Owner,
residual_exact : support in residual
  <-> support in full slice and earlierOwner support = none.
```

`PreC9Owner` has exactly the constructors C1 through C8. Thus every support
selected for the C9 slope cell is formally proved not to have an earlier owner.
This is a finite semantic complement, not an informal adjective.

### Proved payment

For one residual prefix key, the data also carry:

- an `(SE2)` distinct-slope support projection whose supports lie in the
  residual prefix fiber;
- a natural scale; and
- a row-sharp max-fiber theorem on the corresponding full prefix fiber.

The adapter proves

```text
assigned C9 slopes
  <= selected residual supports
  <= residual prefix fiber
  <= full prefix fiber
  <= compilerLoss * naturalScale.
```

It installs the result as a direct C9 `ProfilePayment` and appends it to an
earlier line only after a separate slope-level disjointness proof. The direct
adapter is intentional: once primitive Q / row-sharp max-fiber control is
supplied, inserting a fake Sidon stage would obscure the actual theorem used.

### Scope boundary

The packet does not prove the row-sharp max-fiber inequality. It also does not
construct the concrete RS C1--C8 predicates, prove that the supplied owner
function is the repository's final global atlas, count all realized prefix
keys, establish an independent ray theorem beyond `(SE2)`, or prove a
Fourier/Sidon estimate. It only makes the exact finite post-deletion/max-fiber
producer contract executable and noncircular.

## 4. What this changes in the lane map

### C8

The shallow-prefix route no longer stops at an abstract `DirectRC` proposition:
it now reaches an actual C8 `ProfilePayment` and line-local ledger extension.
The live C8 wall is therefore narrower:

```text
actual first-match chart exhaustion and profile census,
plus deep-prefix MI/MA or another direct payment when shallow closure fails.
```

High shortened-kernel dimension by itself is not the wall.

### C9

The next useful theorem is not another generic low/high-energy partition. It is
one of:

1. a row-sharp primitive prefix max-fiber theorem on the exact residual; or
2. an image-normalized Sidon/MI+MA theorem that implies that max-fiber bound;

followed by a distinct-slope projection such as the supplied `(SE2)`
certificate. The exact earlier-owner complement must remain part of the input.

## 5. Global nonclaims

This packet does **not** prove:

- a fixed-before-line exhaustive C1--C9 atlas;
- nonempty C8 or C9 survival on every line;
- subexponential realized profile count;
- global residual-to-full add-back;
- actual `(UNIF)`;
- identity or target dominance;
- a deployed adjacent-row certificate; or
- final asymptotic RS--MCA resolution.

The modules are intended as a stacked follow-up after the C7/UNIF packet, not
as additions to PR #997.
