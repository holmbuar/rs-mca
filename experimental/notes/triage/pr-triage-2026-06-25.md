2. **M2 exact bridge.** PR #102 records the exact support-wise bridge

   ```text
   epsilon_mca(C,delta)
     = LD_sw(C, ceil((1-delta)n)) / |F|
   ```

   for the same-support two-source object. This is a useful normalization check
   for translating MCA estimates into line-decoding estimates.

3. **F1 extension-line lower floors.** PR #103 upgrades the extension-line
   counterexample note. For fixed rate, the sigma-one family gives the floor

   ```text
   emca(C_F,delta) >= (1-rho)^2/2 - o(1),
   ```

   with higher-slack analogues

   ```text
   emca >= (1-rho)^(sigma+1)/(sigma+1)! - o(1)
   ```

   before extension-degree dilution. This is a stronger warning against naive
   extension-transfer statements, not a positive MCA bound.

4. **M1 beta-pushforward audit.** PR #104 adds a finite spectral audit for the
   beta-pushforward window. It reports no hidden growing `p^2` mass in the
   scanned rows and keeps the remaining conductor-style estimate as an open
   proof target.

5. **L2 sharp target.** PR #107 does not prove a better deployed bound, but it
   sharpens the target: interleaved-list analysis should charge quotient-core
   mass diagonally and avoid the naive Cartesian-product numerator.
