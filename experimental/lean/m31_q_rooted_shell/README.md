# M31 Q rooted-shell Lean package

This is the stdlib-only Lean validation package for the Mersenne-31 rooted-shell
and canonical-padding packets under `experimental/notes/`.

## Scope

The package kernel-checks:

- the generic rooted-shell excess summation compiler;
- the exact `3+7` deployed list/MCA deductions and integer reserves from the
  pinned scaled-floor packet values;
- the finite Chebyshev/twin-coset control showing `2+7` fails while `3+7`
  holds, including support shape, common prefix, distance, quotient-proxy,
  common-core, `T₂`-fiber, and non-product-closure checks;
- the explicit `F_241` order-twenty multiplicative-subgroup packet showing
  that support-level dihedral and planted-core pruning does not imply `3+7`;
- the upstream-#1014 `F_11` padding-bridge counterpacket and exact deployed
  direct-transport blocked bands; and
- the masked diagonal-saturation successor certificate: exact shifted
  last-index floor, prefix-44 cap, first-three padded bounds `20765`, `41530`,
  and `62295`, the `259881` source-key crossing, and typed preservation of
  ordered selectors, root-status masks, semantic owners, refunds, and signed
  occupancy credits.

The padding counterpacket and saturation theorem address different claims.  The
counterpacket refutes direct transport of a named low actual-error basis row.
The successor theorem constructs an intrinsic minimal frame of the full-rank
masked/padded module and therefore leaves the counterpacket intact.

The general polynomial-module statements—diagonal-saturation isomorphism,
successive-minimum monotonicity, predictable degree, and the primitive
46-column Forney sum—are proved in the paired audit notes.  The Lean modules
kernel-check the complete finite counterpacket, deployed arithmetic, and typed
compiler boundary without adding a polynomial-ring or Forney-index library.

## Open boundary

The package does **not** prove:

- existence of a forbidden M31 center;
- common-core add-back or the rank-two coloop terminal;
- an exhaustive semantic C1–C8 owner atlas;
- the open deployed rooted-shell hypothesis or row-sharp Q;
- list-interior coverage, an adjacent safe row, an MCA slope projection, or a
  complete first-match ledger.

The semantic-owner modules queued in upstream PR #1009 and fork PR #64 are
separate and are not modified by the masked-saturation packet.

## Validation policy

There is no Mathlib dependency.  Do not build this package locally in agent
sessions.  Push the complete research packet to a fork branch and open a draft
PR targeting `holmbuar/rs-mca:main`; the repository's PR-triggered GitHub
Actions workflow is the authoritative Lean build.  Green CI proves compilation
only; each correspondence file separately audits the Lean statements against
its source theorem.
